¿Qué es un enlace o URL?


URL significa Uniform Resource Locator (Localizador de Recursos Uniforme). Una URL no es más que una direccion que es dada a un recurso único en la Web. En teoria, cada URL valida apunta a un único recurso. Dichos recursos pueden ser páginas HTML, documentos CSS, imagenes, etc. En la practica, hay algunas excepciones, siendo la más común una URL apuntando a un recurso que ya no existe o que ha sido movido. Como el recurso representado por la URL y la URL en si son manejadas por el servidor Web, depende del dueño del servidor web manejar ese recurso y su URL asociada adecuadamente.

Tipos de URL


HTTP:

La dirección URL cuya sintaxis comienza con HTTP (siglas en inglés de protocolo de transferencia de hipertexto) es la manera estándar de acceder a sitios web con los navegadores que se usan en Internet. Las cuatro partes de una dirección URL HTTP corresponden al esquema, servidor, puerto y ruta, la cual aparecerá la siguiente manera: “scheme://server:port/path”

FTP:

La sintaxis de una dirección URL puede usar el protocolo de transferencia de archivos (FTP, por sus siglas en inglés) para solicitar o enviar documentos desde un servidor web. Este proceso puede lo puede controlar un programa FTP o con un navegador web. La mayoría de los servidores requieren de una contraseña y de un nombre de usuario válido para acceder a ellos. Su sintaxis puede ser algo como: ftp://user:password@server:port/path
Algunos servidores permiten el acceso anónimo, si la dirección URL se escribe de la siguiente manera: ftp://server:port/path

FILE:

La dirección URL que contenga el protocolo FILE sirve para identificar y acceder a una página web, con archivos o documentos en tu computadora. Este protocolo se puede escribir en el navegador web o en el administrador de archivos, como el Explorador de Windows. La dirección URL debe verse como: file://server/path

MAILTO:

Usar una dirección MAILTO URL significa crear un enlace en el que puedas hacer clic e iniciar un mensaje de correo electrónico a alguna dirección especificada por el enlace. La forma más simple de escribir esta dirección URL es: mailto:address

TELNET:

Una dirección que contenga una sintaxis URL TELNET corresponde a un protocolo de red, ya que abre una sesión con un servidor. El puerto predeterminado es el 23 y, por lo regular, se omite, creando un enlace de la siguiente manera: telnet://user:password@server:port/

Extensiones de archivos:


Una extensión de archivo es un sufijo situado al final del nombre de un fichero de computadora. Normalmente le precede un «.» como separación y suele tener una longitud de dos a cuatro caracteres. En sistemas Windows son herederos del antiguo DOS y su objetivo principal es distinguir su contenido de modo que el sistema operativo sea capaz de interpretarlo o ejecutarlo.

Sistemas operativos como Windows utilizan las extensiones de archivo para identificar qué aplicaciones están asociadas con qué tipos de ficheros. En otras palabras, qué aplicación se abrirá al ejecutar el archivo. Por ejemplo, cuando haces doble clic sobre un archivo denominado «prueba_muycomputer. jpg» el sistema operativo buscará cualquier aplicación asociada con los archivos JPG, abrirá esa aplicación y cargará el archivo.

Tipos de extensiones:

  • EXE: uno de los formatos de archivos ejecutables utilizado para que los programas de Windows se puedan ejecutar.
  • PNG: formato de archivo de imagen sin pérdida.
  • DOC / DOCX: archivo de documentos del procesador de textos Microsoft Word.
  • HTM / HTML: el formato de lenguaje de marcado de hipertexto para crear páginas web en línea.
  • PDF: formato de almacenamiento para documentos digitales desarrollado inicialmente por Adobe.
  • MP3: formato de audio digital comprimido.
  • TXT: Archivo de texto ASCII en texto plano.
  • MPEG: archivo de vídeo comprimido
  • COM: aplicación de MS-DOS
